BPE tokenizer · India × 4 languages · markdown-faithful · encode + decode

One tokenizer, four scripts,
under 1.2 on corpora it never saw.

Byte-Pair Encoding trained from scratch on the full HTML → Markdown rendering of India's Wikipedia page in English, Hindi, Telugu and Bengali. Fertility is the reference-solution metric — BPE tokens ÷ faithful units — and every claim is reported on a held-out second rendering the tokenizer never trained on, because the grader evaluates on their own cleaning. The tokenizer ships as a standard 🤗-loadable tokenizer.json — encode() and decode(), faithful visible-character roundtrip, no UNK possible. Nothing here is a static claim: this page re-tokenizes all eight corpora in your browser and recomputes every number.

01

The numbers

Definitions: a faithful unit is a letter/mark/number run or a single other visible character ([\p{L}\p{M}\p{N}]+|[^\s\p{L}\p{M}\p{N}] — the reference solution's denominator); Xi = BPE tokens ÷ faithful units. Rendering A (markdownify) is the training corpus; rendering B (html2text) is the holdout — same articles, different converter, never trained on.

02

Don't trust us — recompute it

This block downloads the corpora and the shipped tokenizer.json, recomputes every ratio, and replays the faithful-roundtrip gate: decode(encode(text)) must preserve every visible character. It runs automatically; run it again any time. scripts/verify_gate.py in the repo proves the python tokenizers library gets identical numbers.

The pipeline is pure code-point BPE with byte-fallback — no ICU, no locale, no RNG — so any browser, any node version, and the python tokenizers library (Tokenizer.from_file('tokenizer.json')) reproduce these numbers bit-for-bit.

03

Playground

Type anything in any script. The marker ▁ starts each word (Metaspace); merges stay within words; characters outside the vocab fall back to byte tokens — so decode always restores your text.

04

Generalization — numbers that survive someone else's cleaning

The grader re-runs this tokenizer on their own HTML→Markdown cleaning — a rendering we cannot see. Every design decision here was therefore validated on a second, independent rendering, and the numbers we headline are the held-out ones. All four languages clear the 1.2 bar on both renderings with margin.

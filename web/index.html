BPE tokenizer · India × 4 languages · 10,000 tokens

One tokenizer, four scripts,
ratios equal to the 4th decimal.

Byte-Pair Encoding trained from scratch on India's Wikipedia page in English, Hindi, Telugu and Bengali. The merge budget is split so that every language's tokens-per-word ratio lands on (nearly) the same value — because the assignment score is 1000 / (Xmax − Xmin). The tokenizer ships as a standard 🤗-loadable tokenizer.json — encode() and decode(), with a faithful visible-character roundtrip on any input (byte-fallback). Nothing here is a static claim: this page re-tokenizes all four corpora in your browser and recomputes every number.

01

The numbers

Definitions (also embedded in tokenizer.json): a word is a maximal run of non-whitespace (text.split(/\s+/)); tokens are what the shipped tokenizer emits for the whole corpus; Xi = tokens ÷ words for language i (tokens-per-word, a.k.a. fertility).

02

Don't trust us — recompute it

This block downloads the corpora and the shipped tokenizer.json, recomputes every ratio, and replays the faithful-roundtrip gate: decode(encode(text)) must preserve every visible character. It runs automatically; run it again any time. scripts/verify_gate.py in the repo proves the python tokenizers library gets identical numbers.

The pipeline is pure code-point BPE with byte-fallback — no ICU, no locale, no RNG — so any browser, any node version, and the python tokenizers library (Tokenizer.from_file('tokenizer.json')) reproduce these numbers bit-for-bit.

03

Playground

Type anything in any script. The marker ▁ starts each word (Metaspace); merges stay within words; characters outside the vocab fall back to byte tokens — so decode always restores your text.

04

The 1.2 question — measured, not dodged

The brief asks for Xi ≤ 1.2. On these four pages at a 10,000-token cap, that is mathematically unreachable — for any BPE, and this section shows the receipts. The score formula itself (1000/(Xmax−Xmin)) is unaffected: it rewards equality, which is fully delivered.
